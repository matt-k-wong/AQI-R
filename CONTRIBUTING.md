# Contributing to AQI-R

Thank you for taking an interest in contributing! This project is currently in the
**design phase** — no firmware has been written yet. All contributions welcome.

## How to Contribute

### 🐛 Found an Error in the Design?
Open an **Issue** describing:
- Which document or diagram contains the error
- What is wrong
- What it should say / show

### 💡 Have a Design Suggestion?
Open an **Issue** tagged `[design]` with:
- The component or subsystem you're improving
- Your rationale (cost, availability, performance, safety)
- Any relevant datasheets or links

### 🔧 Submitting a Pull Request

1. Fork the repository
2. Create a feature branch: `git checkout -b feature/your-change`
3. Make your changes
4. Commit with a clear message: `git commit -m "docs: fix DS3231 I2C address in BOM"`
5. Push and open a Pull Request

### Commit Message Format

Use conventional commits for clarity:
- `docs:` — Changes to documentation or diagrams
- `hardware:` — BOM or schematic changes
- `firmware:` — Source code changes (once firmware phase begins)
- `fix:` — Correcting an error

## What We're Looking For (Design Phase)

- Corrections to I²C addresses, pin assignments, or power specs
- Alternative component suggestions with rationale and cost impact
- Enclosure design suggestions
- Airflow optimisation ideas (sensor placement, intake/exhaust positioning)
- Safety review feedback
- Schematic capture in Fritzing or KiCad (highly wanted!)

## Code of Conduct

Be respectful. This is a learning and building community. Document the fails.
Share the builds.
